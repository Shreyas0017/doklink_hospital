## ğŸ¯ Key Concepts

### 1. Database-per-Tenant
- Each hospital = separate database
- Complete isolation
- Scalable (can move to different servers)

### 2. JWT Sessions
- No session table needed
- Stateless authentication
- Hospital context in token

### 3. Middleware Protection
- All routes protected by default
- Public routes explicitly allowed
- Auto-redirect to login

### 4. Role-Based Access
- 4 roles: Admin, Doctor, Nurse, Staff
- Extensible for granular permissions
- Admin-only routes enforced
